# 元件命名 Skill - 快速使用指南

## 📥 取得 Skill

1. 從 [團隊共享位置] 下載以下檔案:
   - `SKILL.md` - Skill 主檔案
   - `PAGEs_字典_-_字根字首表.tsv` - 最新的字根字首表

## 🚀 如何使用

### 步驟 1: 開啟 Claude

前往 [claude.ai](https://claude.ai) 或使用 Claude App

### 步驟 2: 上傳檔案

在對話中上傳兩個檔案:
1. `SKILL.md`
2. `PAGEs_字典_-_字根字首表.tsv`

### 步驟 3: 啟動 Skill

輸入:
```
我要使用元件命名 skill,幫我為新元件命名。
```

### 步驟 4: 描述需求

清楚描述你的元件:
```
我需要一個顯示[功能]的[容器類型],可以[操作行為]
```

### 步驟 5: 獲得命名建議

Skill 會提供:
- ✅ 符合規範的命名建議
- ✅ 詳細的命名解析
- ✅ 規則驗證結果
- ⚠️ 如果字根不足,會建議新增

## 💡 使用範例

### 範例 1: 基本命名
```
👤 我需要一個顯示個股即時報價的卡片元件

🤖 建議命名: Card_IndividualStock_Quote

命名解析:
- Card (前綴): 卡片容器
- IndividualStock (大類別): 個股
- Quote (子類別): 報價

符合規則: ✓
```

### 範例 2: 檢查現有命名
```
👤 我們有個元件叫 Filter_Container_PersonalSelection,這樣對嗎?

🤖 結果: ❌ 不符合規範

問題: 功能(Filter)出現在前綴(Container)之前,違反順序規則
建議修正: Container_PersonalSelection_Filter
```

### 範例 3: 批量命名
```
👤 請幫我命名這些元件:
1. 顯示大盤走勢的圖表
2. 編輯用戶暱稱的輸入框
3. 顯示期貨報價的列表

🤖 [會提供所有元件的命名建議]
```

## 📋 命名規則快速參考

### 分類順序 (必須遵守)
```
[前綴] → [大類別] → [子類別] → [功能] → [詳細描述]
```

### 唯一性規則
- 每個分類層級最多出現一次
- 同一層級可以有多個字根 (如: 兩個大類別)

### 常見錯誤
❌ `Filter_Container_Tab` - 順序錯誤
✅ `Container_Tab_Filter` - 順序正確

❌ `Row_Edit_Search_Filter` - 功能重複
✅ `Row_Filter` - 選擇主要功能

## ❓ 常見問題

**Q: 我一定要上傳 SKILL.md 嗎?**
A: 是的,這是 Claude 理解命名規則的關鍵檔案。

**Q: 字根字首表更新了怎麼辦?**
A: 從共享位置下載最新版本,重新上傳即可。

**Q: 可以跳過某些層級嗎?**
A: 可以。例如 `Card_IndividualStock` 沒有子類別也是正確的。

**Q: Skill 的建議一定是對的嗎?**
A: Skill 會依據規則給建議,但最終仍需團隊成員判斷是否符合業務語境。
